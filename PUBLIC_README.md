# Footprint Public Template

Footprint Public Template 是一个纯前端个人旅行足迹模板，适合用来记录和展示自己的酒店、城市、铁路与航班/路线足迹。

## 特点

- 纯前端项目，可以直接打开 `app.html` 使用，也可以部署到 GitHub Pages。
- 包含酒店足迹、城市足迹、铁路足迹、航班足迹四个模块。
- 支持列表、筛选、搜索、排序、地图 marker、路线直线、编辑、新增、删除、导入导出和本地备份。
- 顶部工具区提供开始使用引导、网页内帮助中心、完整备份、示例数据管理、CSV 模板与导入、缺坐标管理。
- CSV 和缺坐标管理使用独立面板，避免挤在左侧筛选栏中。
- 默认只提供少量虚构示例数据，不包含任何真实个人出行记录。

## 数据保存

本项目是纯前端应用，默认不会上传用户数据到服务器；数据主要保存在当前浏览器本地，用户需要自行导出备份。浏览器中的新增、编辑和删除通常保存在本地 localStorage。换浏览器、清理缓存或更换设备时，本地数据可能丢失。

请定期使用页面内的“导出完整备份”保存数据；完整备份包含酒店、城市/目的地、铁路、航班/路线和本地补充坐标。部署前请确认要发布的数据已经脱敏。


## 首次使用

首次打开 `app.html` 时，页面会显示“开始使用”引导，说明这是纯前端应用、数据默认保存在当前浏览器本地、不会上传用户数据到服务器，并提醒定期导出完整备份 JSON。

你可以使用“一键清空示例数据”把酒店、城市、铁路和航班示例清空。清空状态会写入当前浏览器本地，刷新后仍保持为空。需要重新查看模板效果时，可以使用“恢复示例数据”回到项目内置示例。关闭引导后刷新不会重复弹出，也可以在顶部工具区重新打开。

## CSV 导入

在顶部工具区打开“CSV 模板与导入”，可以分别下载酒店、城市/目的地、铁路、航班四类 CSV 模板。填写后可导入到对应模块，并选择追加到当前数据或覆盖当前模块数据。

CSV 建议使用 UTF-8 编码保存。导入时会检查表头和必填字段，并尽量把 `lat`、`lng`、`price`、`distanceKm` 等数字字段转为数字。空字段可以留空。

## 缺坐标管理

导入数据后，如果酒店、城市、铁路车站、机场或港口缺少坐标，可以在顶部工具区打开“缺坐标管理”，刷新列表并手动补充。补充坐标会保存在当前浏览器 localStorage，不会直接修改 `rail-stations.js` 或 `flight-airports.js`。

缺坐标管理内置坐标助手：

- 复制搜索关键词：按酒店、城市、车站、机场或港口自动生成适合搜索的关键词。
- 打开地图搜索：打开 OpenStreetMap 或 Google Maps 的普通搜索页面，不接入地理编码 API，也不会上传到 Footprint 服务器。
- 粘贴坐标自动识别：支持 `31.2304, 121.4737`、`121.4737, 31.2304`、`lat:31.2304 lng:121.4737` 等格式。
- 从地图点选：在面板内小地图上点击位置，自动填入 lat / lng，保存前可以手动微调。

建议使用 WGS84 / OpenStreetMap / Google Maps 坐标。国内地图如高德、百度、腾讯复制的坐标可能存在偏移，如发现位置不准，请用地图点选方式微调。

地图渲染时会优先使用本地补充坐标，再使用内置坐标文件。完整备份 JSON 会包含本地补充坐标；也可以单独导出本地补充坐标 JSON。

## 网页内帮助中心

页面顶部工具区提供“帮助中心”入口。帮助中心默认面向普通用户，包含快速开始、新增/编辑/删除、CSV 导入、完整备份、数据保存与隐私、缺坐标处理和常见问题。GitHub Pages / CloudBase 放在底部“进阶：自托管部署”折叠区，普通使用无需阅读。

## 部署

如果你想部署自己的 Footprint 副本，可以把整个目录发布为静态网站。

- GitHub Pages：把项目文件提交到 GitHub 仓库并启用 Pages；根路径 `index.html` 会自动进入 `app.html`。
- CloudBase：按静态网站上传整个目录即可，不需要后端、数据库或账号系统。

## 如何替换示例数据

你可以直接编辑以下文件，也可以先在页面中编辑后导出：

- `hotels.js`：酒店示例数据
- `places.js`：城市/目的地示例数据
- `rail-trips.js`：铁路行程示例数据
- `rail-stations.js`：铁路车站坐标
- `flights.js`：航班/路线示例数据
- `flight-airports.js`：机场/路线点坐标

请不要把身份证号、订单号、票号、二维码、乘客姓名、邮件原文、截图或 Excel 原始导出文件提交到公开仓库。

## 本地检查

修改 JS 后建议运行：

```bash
node --check app.js
node --check hotels.js
node --check places.js
node --check rail-trips.js
node --check flights.js
```
